# Harpreet's Project Understanding (Code Reviewer)
Last Updated: 2026-05-08 00:35

## Review Standards for This Repo
- **Linting**: Ruff via `make lint`. CI fails on lint violations. Reject PRs that don't pass.
- **Formatting**: Ruff format via `make format`. Enforce.
- **Python version**: pinned to `==3.12.*`. Reject syntax/typing requiring 3.13+.
- **Pydantic version**: `~=2.9`. Don't use deprecated v1 patterns; don't pre-adopt v3 patterns. `Field(..., example=...)` is deprecated — use `json_schema_extra`.
- **Type hints**: Pervasive in this codebase. New code without hints is a regression — flag it.
- **Async correctness**: FastAPI routes are async; don't block the event loop with sync DB calls. Use the async `psycopg`/`psycopg-pool` patterns already established.

## Architectural Boundaries (Reject Violations)
- **Routers stay thin** — business logic belongs in `src/services/`, not in routers. New endpoints with embedded SQL/LLM calls in the router are a `[REVIEW] reject`.
- **DB access goes through `src/dependencies/` clients** — direct `psycopg.connect()` from random services is a smell.
- **No new endpoints in `app.py`** — must use a router under `src/routers/`.
- **Schemas in `schemas.py`, models in `models.py`** — don't scatter Pydantic classes across services.
- **`_call_llm_json` is the LLM-as-JSON gateway** — direct `openai.chat.completions.create(...)` calls in new services are a smell unless there's a clear reason.

## Quality Hot Buttons (project-specific)
- **Pydantic silent-drop**: new extraction code MUST have per-shape tests on the producer side. Reject PRs adding extraction fields without shape tests.
- **Silent default-to-0**: `.get("cost_usd", 0)` style defaults in measurement/billing code should be compute-or-warn instead. Flag.
- **Fail-closed contracts**: any validation that drops/skips on failure must have producer tests proving the producer actually emits valid shapes. Otherwise we get silent skips in prod.
- **Cache invalidation**: any write that changes profile/portfolio/intents must bump the Redis namespace. Flag if missing.
- **Migration safety**: migrations must be idempotent and reversible where possible. New `CREATE TABLE` without an `IF NOT EXISTS` or rollback is a flag.
- **Secrets in code**: gitleaks runs in CI. Reject anything that smells like a credential. Allowlist is in `.gitleaks.toml`.

## Existing Code Smells (Pre-existing, not blockers but worth noting)
- Three coexisting prompt files (`prompts.py`, `prompts_v2.py`, `prompts_v3.py`)
- Split-brain `src/storage/` vs `src/services/storage.py`
- `app.py` still hosts many legacy endpoints not yet routed through `src/routers/`
- Test files at repo root (`test_profile_extraction.py`, etc.) outside the `tests/` tree
- These should be deprecated/cleaned up in dedicated stories, not silently cleaned up in feature PRs.

## Review Output Format
- For approve: `[REVIEW] approved` to bus, set status `review` -> `testing`.
- For reject: `[REVIEW] rejected` with file:line specifics, what to fix, and why.
- Track review cycle in status.md (max 3, then escalate).
