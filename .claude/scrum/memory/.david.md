## Patterns to Follow
- New endpoints go in `src/routers/{domain}.py`, mounted from `app.py`. Don't add new endpoints directly to `app.py`.
- For LLM-as-JSON, call `src/services/extract_utils.py::_call_llm_json`.
- For LangGraph extraction, add a node to `unified_ingestion_graph.py` rather than forking.
- For caching, use the Redis namespace-bump pattern (see profile cache).
- Pydantic schemas go in `src/schemas.py`; data models in `src/models.py`.
- Prefer existing service modules over creating new ones.

## Footguns (lessons inherited)
- **Pydantic silently drops unknown keys**: `"name"` vs `"asset_name"` will pass validation and lose data. Add per-shape tests when changing extraction output.
- **Silent `dict.get(key, 0)` defaults swallow measurement bugs** — for measurement keys, prefer compute-or-warn over default-to-0.
- **Fail-closed contracts need producer tests**: pair every fail-closed validation with per-producer shape tests.
- **JS/UI cache busting**: HTML-pinned `<script src="js/foo.js?v=N">` means after editing JS you must bump the `?v=N` or browsers run stale code.
- **`CHROMA_HOST` baking**: stale env can get baked into the API container — `--force-recreate` to fix.
- **Three prompt files** (`prompts.py`, `prompts_v2.py`, `prompts_v3.py`) — read carefully which is being imported before editing.

## Langfuse SDK (v3 / langfuse==3.14.2) — usage rules
- Wrap form (sync): `with client.start_as_current_observation(as_type="span", name=..., trace_context={"trace_id": ..., "user_id": ..., "session_id"?: ...}, input=..., metadata=...) as root_span: ...`.
- Generate trace ids with `client.create_trace_id()` (don't fabricate).
- Detect if a parent span is already active: `client.get_current_trace_id()` returns `Optional[str]` (None if no active span). There is **no** `get_current_span()` in v3.
- On success: `root_span.update(output={...small summary...})`. Keep payloads small — counts, ids, latencies. **No** embedding vectors, **no** full retrieval bodies.
- On exception: just let the `with` exit; the context manager closes the span. Re-raise (or let it propagate).
- Always degrade gracefully: import errors / client failures must not break the underlying call. Mirror `unified_ingestion_graph.py:1468-1480`.

## Conventions / gotchas (Langfuse-specific)
- Existing `tracing.start_span(...)` calls in `compaction_graph.node_*` and elsewhere are **child** spans only — they need a root span in the contextvar to attach to. Once we create a real `start_as_current_observation` root via the singleton client, those nested `start_span` calls (which read `_current_client`/`_current_root_span` from `tracing.start_trace`) will NOT auto-attach unless we also call `start_trace`. BUT they'll still chain via OTEL because `start_span` itself uses `client.start_as_current_observation` which inherits OTEL context. So nested `start_span` calls in the graph body will attach as OTEL children of our outer wrap even though the `_current_root_span` contextvar isn't populated. Verified by reading `tracing.py` and the SDK pattern.
- For compaction: replacing `start_trace` with `with langfuse_client.start_as_current_observation(...)` means `_current_root_span` is no longer set by `tracing.py`, so `start_span` (which checks `_current_client.get()` and `_current_root_span.get()`) returns None for nested spans. Workaround: also populate `_current_client` and `_current_root_span` so existing nested spans keep working — OR migrate them. Cheapest = populate the contextvars manually around the with block.
- Direct ingest endpoint: `POST /v1/memories/direct` (router prefix `/v1/memories` + route `/direct`).
- Always use the singleton via `from src.dependencies.langfuse_client import get_langfuse_client`. Don't instantiate `Langfuse(...)` directly except in `unified_ingestion_graph.py` (existing smell, separate story).
