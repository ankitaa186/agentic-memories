## Codebase Layout (where things live)
- **HTTP layer**: `src/app.py` (legacy + lifespan) + `src/routers/{profile,portfolio,intents,memories}.py`
- **Orchestration**: `src/memory_orchestrator/` (`orchestrator.py`, `ingestion.py`, `retrieval.py`, `client_api.py`, `policies.py`)
- **LangGraph pipelines**: `src/services/unified_ingestion_graph.py`, `src/services/compaction_graph.py`
- **Domain services**: `src/services/*.py` — episodic, emotional, procedural, persona, profile, portfolio, intent, summary, reconstruction, hybrid_retrieval, etc.
- **DB clients**: `src/dependencies/{chroma,timescale,redis_client,langfuse_client}.py`
- **Auth (CF Access)**: `src/dependencies/cloudflare_access.py`
- **Schemas**: `src/schemas.py` (Pydantic)
- **Models**: `src/models.py`
- **Migrations**: `migrations/{postgres,timescaledb,chromadb}/`; archived neo4j/old_v1
- **Tests**: `tests/{unit,integration,e2e,memory_orchestrator,evals}/` — pytest

## Dev Commands (use these, not ad-hoc)
- Setup: `make install` (uses `uv` if available)
- Run stack: `make start` (or `docker compose up -d --build api`)
- Stop: `make stop`
- Tests: `make test-fast` for quick feedback; `make test-unit` / `test-integration` / `test-e2e`; full `make test`
- Lint: `make lint`; format: `make format` (Ruff)
- Migrations: `make migrate` (uses `POSTGRES_PASSWORD` from `.env`)
- Rebuild API only: `docker compose up -d --build api`
- Python deps: `uv pip install <pkg> --python .venv/bin/python`

## Patterns to Follow
- New endpoints go in `src/routers/{domain}.py`, mounted from `app.py`. Don't add new endpoints directly to `app.py`.
- For LLM-as-JSON, call `src/services/extract_utils.py::_call_llm_json`.
- For LangGraph extraction, add a node to `unified_ingestion_graph.py` rather than forking.
- For caching, use the Redis namespace-bump pattern (see profile cache).
- Pydantic schemas go in `src/schemas.py`; data models in `src/models.py`.
- Prefer existing service modules over creating new ones.

## Footguns (lessons inherited)
- **Pydantic silently drops unknown keys**: `"name"` vs `"asset_name"` will pass validation and lose data. Add per-shape tests when changing extraction output.
- **Silent `dict.get(key, 0)` defaults swallow measurement bugs** — for measurement keys, prefer compute-or-warn over default-to-0.
- **Fail-closed contracts need producer tests**: pair every fail-closed validation with per-producer shape tests.
- **JS/UI cache busting**: HTML-pinned `<script src="js/foo.js?v=N">` means after editing JS you must bump the `?v=N` or browsers run stale code.
- **`CHROMA_HOST` baking**: stale env can get baked into the API container — `--force-recreate` to fix.
- **Three prompt files** (`prompts.py`, `prompts_v2.py`, `prompts_v3.py`) — read carefully which is being imported before editing.

## Branch / Workflow
- Feature branch per story: `git checkout -b story/{N}.{M}-{short-desc}`
- Don't push without instruction
- Always run `make test-fast` and `make lint` before posting `[STATUS]` review-ready
- Note: `tests/test_*.py` at repo root and `test_profile_extraction.py`, `test_request.json`, `test_orchestrator_scenario.sh` exist outside `tests/` — be careful not to mistake them for the canonical suite (the canonical suite is in `tests/`).
